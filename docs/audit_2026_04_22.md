## Security posture

- No `eval` / `exec` / `pickle.load` / `subprocess` usage
- No mutable default arguments
- All `requests.get` calls have explicit timeouts
- All `except Exception` blocks either log the error or document why they swallow silently
- `unsafe_allow_html=True` usages now HTML-escape variable interpolations
- SEC-compliant User-Agent on every EDGAR request
- Token-bucket rate limiting shared across every EDGAR caller
- `EDGAR_CONTACT_EMAIL` placeholder guard prevents accidental deploy without identification

## Known residuals (not fixed ‚Äî flagged for future)

- `integrations/broker_mock._apply_slippage` uses unseeded `random` ‚Äî non-deterministic fills, documented, acceptable for demo
- `core/signal_adapter._phase1_fallback` thresholds (ret/vol ratio > 0.70 for BUY) put almost every category-default ETF into HOLD ‚Äî conservative by design but could use tuning post-demo
